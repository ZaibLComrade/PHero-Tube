Q: Discuss the scope of var, let and const.

=> var, let and const are keywords used to declare variables in JavaScript. Among which, var is the oldest way to declare variables. var has a scope of functional and global level.Variables declared with var keyword outside any function are global-scoped meaning they are accessible throughout any part of the code. whearas if declared within a function, the variable becomes accessible from any part of the function but not outside the function-scope. var keyword hoists the variable to the top of it's scope initializing with the value "undefined" making the variable accessible even before declaration"

let and const keywords were introduced in ES6 which are comparetively new. Both has the scope of block-level. variables declared with let or const are accessible from only the code block. The block can be functions, if...else statements or loops. They can also be global-scoped. The outer scope of the block does not have access to the variables. Both let and const keywords are hoisted at the top of the block too. But are not initialized before declaration which causes in a reference error when trying to access before declaration.

Q: Tell us the use cases of null and undefined

=> null and undefined are both used to indicate absence of value in JavaScript. While undefined is the default implicit value of a variable without a value, null is more of an inteded value for variables that should not have any value. undefined is mostly usded as a value by the program for uninitialized variables, functions that are called with fewer arguments than it is expected or when functions do not have an explicit return or accessing object properties that do not exist.

Despite of having fewer distinction with undefined, null is used to intentionally indicate the emptiness of a variable. It's also used to reset variable values when the variable sould not contain any value.

Q: What do you mean by REST API?

=> REST API or Representational State Transfer Application Programming Interface is a set of rules and conventions for interactions of web services as well as for building them. REST is an architectural style that simplifies software components communication over the internet. Resources are represented as objects in REST and standard HTTP methods like GET, POST, PUT and DELETE are used to perform actions on these resources. Data is often exchanged in formats like JSON and XML. The request from client to server contains all necessary information to understand the request which simplifies the process. HTTP status code like 200(OK), 404(Not found) are used to indicate the outcome of it's request. The main reason for its usage in web development is it's simplicity and scalability, as well as the fact that the APIs are accessible by various clients including web browsers, mobile apps and other software applications.
